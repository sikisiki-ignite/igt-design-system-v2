02 · Hierarchy Map — AI 우선순위 기준
한 화면에서 사용 레벨이 높을수록 해당 토큰 사용 빈도는 낮아야 한다. 레벨이 낮은 토큰이 UI의 대부분을 채운다.
1
display-lg / md
Hero, 랜딩 최상단 — 한 페이지에 최대 1회. KPI 대형 숫자.
2
display-sm
페이지 제목 (h1 수준). 설정/대시보드 진입 섹션 헤더.
3
heading-lg
모달 제목, 사이드 패널 헤더, 탭 섹션 최상위 제목.
4
heading-md
카드 그룹 헤더, 폼 섹션 구분자 (h2 수준).
5
heading-sm
개별 카드 제목, 사이드바 섹션명, 테이블 캡션 (h3 수준).
●
body-md
기본값. body 태그 전체. 모달 설명, 단락 텍스트.
6
body-sm
카드 내부 설명, 목록 항목, 툴팁 본문.
7
body-xs
테이블 셀, 인풋 값, 드롭다운 아이템 — 밀집 레이아웃.
8
label-md / sm / xs
버튼, 폼 라벨, 탭, 뱃지, 테이블 헤더 — 인터랙티브 요소.
9
caption-md / sm
도움말, 에러, 타임스탬프 — 화면 하단부 보조 정보.

04 · AI Quick Reference — 컴포넌트별 토큰 매핑
컴포넌트 / 요소 토큰 주의
Page title (h1) display-sm 대시보드·설정 최상단 1회만
Modal / Drawer title heading-lg
Card title heading-sm 카드 내부 h3 수준
Section divider heading-md 폼 / 콘텐츠 그룹 구분
Button (lg) label-md
Button (md, 기본) label-sm 버튼 기본 크기
Button (sm) label-xs
Form label label-sm
Input / Select value body-xs placeholder 동일 토큰, 색상만 muted
Helper / Error text caption-md error: danger 색, helper: ink-3
Table header (th) label-xs uppercase + letter-spacing 권장
Table cell (td) body-xs
Tab label label-sm
Badge / Tag label-xs or caption-md label-xs: 인터랙티브, caption: 정적 정보
Tooltip caption-md
Toast message body-xs 제목 있으면 label-sm + body-xs
KPI 대형 숫자 display-md 단위/라벨은 label-xs 또는 caption-md
Breadcrumb body-xs current 페이지는 weight 500
Sidebar nav item body-sm or label-sm active: weight 600
Empty state title heading-md 설명: body-sm
Dropdown item body-xs
05 · AI 규칙 — 타이포그래피 사용 원칙
DO ✓
  • ✓ CSS에서 --text-{role}-{size}-{prop} 변수만 참조
  • ✓ font-size, font-weight, line-height를 항상 3개 세트로 적용
  • ✓ body 기본값은 body-md (16px/400/24px) — 이미 body에 설정됨
  • ✓ label 계열은 인터랙티브 요소(버튼·탭·폼 라벨)에만 사용
  • ✓ 한 컴포넌트에서 최대 2~3단계 위계만 사용
DON'T ✗
  • ✗ px/rem 하드코딩 — font-size: 14px 직접 사용 금지
  • ✗ --ref-font-* 프리미티브 토큰을 컴포넌트에서 직접 사용 금지
  • ✗ display 계열을 본문/카드/폼 내부에 사용 금지
  • ✗ font-weight만 임의로 바꿔 강조 — 토큰 role을 전환해야
  • ✗ 한 뷰에 display 3개 이상 중첩 사용 금지
